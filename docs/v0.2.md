# @holmhq/sdk 0.2

`0.2.0` adds a complete, audited admin/operator preview while preserving the
stable web/BFBB contract introduced in `0.1.0`. The release does not remove,
rename, or behaviorally break a stable `0.1.x` entry point.

## Support contract

Stable entry points remain additive throughout `0.2.x`: no breaking removal,
rename, or behavioral contract change. A future removal must be deprecated
first. Preview and reserved entry points may evolve until explicitly promoted.

| Entry point | Status | Contract |
| --- | --- | --- |
| `@holmhq/sdk` | stable | Runtime-neutral core composition and shared types. |
| `@holmhq/sdk/core` | stable | Capabilities, caller identity, lifecycle, errors, diagnostics, cancellation, and wire values without DOM or Node ambient types. |
| `@holmhq/sdk/transports` | stable | Requests, responses, auth application, caller-partitioned caching, uploads, and credential-safe diagnostics. |
| `@holmhq/sdk/app` | stable | Holm app HTTP, auth, links, pagination, uploads, and built-in surface helpers. |
| `@holmhq/sdk/web` | stable | Browser runtime plus `createWebApp()`. It can transport admin requests only for explicit operator callers. |
| `@holmhq/sdk/state` | stable | Framework-neutral query, mutation, derived, history, and reconciliation resources. |
| `@holmhq/sdk/test` | stable | In-memory runtime and deterministic clock utilities. |
| `@holmhq/sdk/admin` | preview | `createAdminClient()`, generated admin namespaces, immutable audited inventory, command/upload/binary seams, and explicit operator caller gating. Not frozen. |
| `@holmhq/sdk/node` | preview | Node adapter and injected operator/token services; usable for bounded integrations, not frozen. |
| `@holmhq/sdk/sobek` | preview | Holm's injected server/Sobek adapter; not frozen. |
| `@holmhq/sdk/bridge` | reserved | Mailbox and service-slot contracts for tests and future native shells, not a production runtime. |

See [capabilities.md](capabilities.md) for product-level availability and
[admin.md](admin.md) for operator composition and migration details.

## Admin scope

The admin route ledger is pinned to a named Holm commit and contains:

- 174 audited inventory keys;
- 189 expanded HTTP route/method contracts;
- 216 unique generated SDK methods; and
- 18 intentional exclusions covering browser redirects, local development
  helpers, public ingress, and internal transport/workflow routes.

Every generated method is source-linked through immutable descriptors. The
generator/drift gate fails if method declarations or descriptors diverge from
the committed ledger. Methods use one standardized operation object for path,
query, body, headers, sensitivity, cancellation, binary mode, command arguments,
and runtime-neutral uploads.

`createAdminClient()` requires an explicit runtime and caller. Web and Node
adapters advertise `holm.http.admin`, reject non-operator caller context before
network I/O, and keep cookie/token proof private. This is defense in depth and
client isolation—not authorization. Holm remains authoritative for every route.

The admin API is preview because the operation-input ergonomics have not yet had
a full external operator-client compatibility cycle. It passed the same source,
type, declaration, generated-artifact, package, size, reproducibility, and
review gates as stable code.

## Distribution

Install the immutable npm release:

```bash
npm install @holmhq/sdk@0.2.0
```

BFBB/operator clients may vendor the complete `dist/` module tree from the
immutable npm version, Git tag `v0.2.0`, or an exact reviewed Git SHA. The
complete `dist/holm.js` composition includes preview admin; the narrow
`dist/holm-web.js` app composition deliberately excludes it. Verify
`dist/manifest.json` and never deploy from mutable `@main`.

## Compatibility and migration

- Stable `0.1.x` entry points remain source- and declaration-compatible.
- `@holmhq/sdk/admin` is additive and isolated from root/stable declaration
  graphs.
- Admin core compiles without DOM or Node ambient types.
- The legacy Holm `packages/holm-sdk` admin client remains live; this release
  does not delete, redirect, or deprecate it.
- Upload convenience is intentionally redesigned around an injected shared
  `UploadRequest` service rather than universal `Blob`, `FormData`, filesystem,
  or ZIP assumptions.
- Attachment downloads return SDK `ReadonlyBytes`; command helpers retain
  Holm's canonical `/api/cmd` success/error envelope semantics.

## Evidence required for release

A releasable `0.2.0` candidate must have all of the following at one reviewed
commit:

- deterministic admin inventory generation and complete parity tests;
- core/type/source/declaration/dist/example/package gates green;
- measured coverage, size, licenses, and reproducibility green;
- an installed-tarball import smoke for every package entry point;
- one independent SDK review with no unresolved P1/P2 findings; and
- fresh read-only Holm-authority acceptance against a named current commit.

Publication, tag, and release assets occur only after these gates pass. npm and
Git release versions are immutable once published.
